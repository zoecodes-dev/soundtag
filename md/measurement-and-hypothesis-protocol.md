# 측정 체계와 가설 프로토콜 — SoundTag

> 이 문서는 "좋아 보이는 validation accuracy를 의심하고, 측정 체계 자체의 결함을
> 진단해 방법론을 재설계한 과정"의 기술적 증거다. 프로젝트의 중심 서사를 떠받친다.

---

## 1. source-type leakage — 무엇이고 어떻게 발견했나

### 1-1. 증상

장르 분류 모델을 5번 학습했다. 5번 모두 검증 정확도를 *학습 데이터와 같은 출처에서
나눈 validation split*으로만 측정했다. 가장 높았던 점수는 hybrid v1의 검증 정확도
**77.6%**였다.

표면적으로는 "괜찮은 baseline"으로 보였다. 하지만 한 가지가 계속 걸렸다 —
**이 점수는 K-pop에 대한 성능이 아니었다.** 타겟 도메인(K-pop 데모)에서 모델이
실제로 얼마나 맞히는지는 단 한 번도 측정한 적이 없었다.

### 1-2. 진단: 모델은 장르가 아니라 "출처"를 학습했다

Hybrid v3 데이터셋은 두 출처를 합친 것이다.

| 출처 | 포맷 | 특성 |
|---|---|---|
| FSLD | wav | 전문가 태깅, 무손실, 길이/녹음 환경 일관 |
| Freesound | mp3 | 사용자 업로드, 손실 압축, 길이·게인·노이즈 제각각 |

같은 장르라도 두 출처는 **인코딩 아티팩트·주파수 롤오프·길이 분포·노이즈 플로어**가
서로 다르다. 모델 입장에서는 이 출처 신호가 장르 신호보다 *학습하기 쉽고 분리도 더
잘 되는* 단서였다. 즉 모델은 "이건 Trap이다"가 아니라 "이건 FSLD에서 온 파일이다"를
학습하고 있었다 — 이것을 **source-type leakage**라 부른다.

validation split이 학습셋과 *같은 출처 분포*에서 나왔기 때문에, leakage가 있어도
검증 정확도는 높게 나온다. 77.6%는 모델이 장르를 이해해서가 아니라, 출처를 구분해서
얻은 점수였다. **가짜 신호.**

### 1-3. 근본 원인: 측정 체계의 결함

핵심 통찰은 "모델이 약하다"가 아니라 **"측정이 잘못됐다"**였다.

- 5번의 실험이 전부 *in-distribution*(학습 분포 내부)에서만 평가됐다.
- 타겟 도메인인 **실제 K-pop으로 측정한 적이 한 번도 없다.**
- 그래서 source-type leakage 같은 분포 밖 실패를 구조적으로 감지할 수 없었다.

→ 5번의 결과는 실패 모음이 아니라, **"공통 원인을 가진 5개의 데이터 포인트"**다.
공통 원인(leakage + 측정 부재)을 진단하고 나서야 방법론을 다시 설계할 수 있었다.

---

## 2. 해결책 (1): 출처 분리 — dual-model

한 모델에 출처가 다른 오디오를 섞는 한, leakage는 반복된다. 그래서 **입력 종류별로
모델 자체를 분리**했다.

- **Model A — 드럼루프 전용.** 추론 시 입력곡을 Demucs로 분리한 `drums` stem만 받는다.
  드럼 사운드만 남기므로 풀믹스 출처 특성에 휘둘리지 않고, 리듬·그루브에 집중한다.
- **Model B — 풀트랙 전용(planned).** 풀믹스 전체를 받아 화성·음색·편곡 차원을 담당한다.

두 입력을 하나의 모델에 넣으면 다시 "장르가 아니라 입력 종류"를 학습하게 되므로,
분리는 leakage를 구조적으로 차단하는 설계다.

---

## 3. 해결책 (2): K-pop hold-out 평가셋

측정을 고치는 핵심은 **학습에 한 번도 쓰지 않은 실제 K-pop**으로만 평가하는 것이다.

### 3-1. 설계 원칙

- **출처 독립** — 학습 데이터(FSLD/Freesound 드럼루프, Deezer 풀트랙)와 완전히 분리된
  실제 K-pop 트랙. 학습에 절대 포함하지 않는다.
- **section-level annotation** — 곡 전체에 라벨 하나가 아니라, intro/verse/pre/chorus/
  bridge 등 **구간 단위**로 라벨링한다. K-pop은 한 곡 안에서 장르가 바뀌므로
  (verse는 R&B, chorus는 EDM 등) 곡 단위 라벨은 측정 해상도를 떨어뜨린다.
- **100곡 분포** — 메이저 그룹/솔로를 아우르는 다양한 스타일로 구성.
  (현재 `data/kpop_ground_truth_template.csv`에 후보 트랙 목록과 라벨 컬럼 골격이
  준비됨. **라벨링은 in progress — 가이드/템플릿 단계.**)

### 3-2. 구간별 라벨 스키마

| 필드 | 값 | 설명 |
|---|---|---|
| `genres` | 1–3개 | 해당 구간의 장르(다중 가능, K-pop은 multi-label) |
| `energy` | 1–5 | 구간 에너지 강도 |
| `mood` | vocabulary | 통제된 무드 어휘 집합에서 선택 |

> 다중 장르(1–3개)를 허용하는 이유: "K-pop은 모든 장르를 가져다 쓴다"는 도메인 특성상
> 단일 라벨 강제는 ground truth 자체를 왜곡한다.

## 4. 향후 모든 실험의 프로토콜

leakage 사례에서 얻은 교훈을 절차로 고정한다. **모든** 실험은 다음을 따른다.

1. **가설 문서화 (1페이지)** — 실험 전에 작성.
   - 검증하려는 **falsifiable hypothesis** (반증 가능한 형태로)
   - 예상 결과와 **실패 기준**(어떤 결과가 나오면 가설을 버리는가)
   - 측정 지표와 평가셋(반드시 K-pop hold-out)
2. **24h quick test** — 풀스케일 학습 전에, 24시간 내 끝나는 소규모 실험으로
   가설을 먼저 친다. 빠르게 죽일 가설은 빠르게 죽인다.
3. **K-pop hold-out으로만 측정** — in-distribution validation 점수는 *참고용*일 뿐,
   판단 근거로 쓰지 않는다. 채택/기각은 hold-out 성능으로만 결정한다.
4. **confusion matrix 분석** — 단일 정확도 숫자로 끝내지 않는다. 어떤 장르가 어떤
   장르로 헷갈리는지, 출처/구간별로 오류가 쏠리는지 행렬로 본다. (77.6%가 가짜였던
   것처럼, 집계 숫자는 실패 구조를 숨긴다.)

---

## 5. 이 문서가 증명하는 것

- 높은 검증 정확도를 **결과로 받아들이지 않고 의심**했다.
- 의심을 **재현 가능한 원인 진단**(source-type leakage)으로 바꿨다.
- 진단을 **측정 체계 재설계**(dual-model + K-pop hold-out + 가설 프로토콜)로 연결했다.

즉 "모델을 5번 돌려 77.6%를 얻었다"가 아니라, **"77.6%가 왜 가짜인지 밝히고, 다시는
같은 방식으로 속지 않을 측정 체계를 만들었다"**가 이 프로젝트의 결과물이다.
